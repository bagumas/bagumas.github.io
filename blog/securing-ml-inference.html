Securing ML Inference: From Model Encryption to Guardrails

This post summarizes a hardened inference stack: AES‑GCM encrypted models, API‑key auth, OAuth on the web app, and adversarial testing in CI using ART.

  • Encrypt models at rest; decrypt only in memory or temp file at runtime.
  • Gate inference with API keys (or mTLS/OAuth) and strong rate limits.
  • Add prompt/IO guardrails for LLMs; sanitize outputs before rendering.
  • Automate adversarial tests (FGSM/PGD/CW) and fail builds on regressions.
  • Monitor drift, anomalies, and extraction/inversion indicators.
